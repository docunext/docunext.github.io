What's up with __call__?

I finally figured out what's up with call. Imagine you have a class. You create an object instance of it. class car, my_car = car(), and when my_car is created, init is called. If you want to run my_car as a function, you can call my_car(), but you'll need a call function. I'm not really sure what the benefit of this is, but I imagine its good for something.
